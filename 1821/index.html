Problems copying large files

June 15, 2021 by pelosnecios

I am having problems using PHP’s copy with large files (>14 GB). The input and output files are the same size, but it seems it can only copy a small portion (up to 1,048,576 bytes, or 1024×1024 bytes) correctly and the rest is filled with garbage data.

This happens on both PHP 7.4.20 and 8.0.7 (64-bit) on Windows/NTFS.

Does anybody have any clue on why is this ocurring? Should I expect file copy function to properly copy a file regardless of size or should I put additional checks around it?

Thanks for your help.

    > should I put additional checks around it?

    If you care about the data, you should always check it. Even if your the software is perfect (doesn’t seem to be in your case), drives go bad all the time.

    If SHA1 is taking too long, you could use CRC. In a security context (making sure a file hasn’t been tampered with) CRC is unacceptable, but for detecting ordinary corrupt data it’s fine and much much faster.

    You could try copying using Windows XCOPY:

    @ECHO ===COPYING VIDEOS > e:Backup.txt
    XCOPY “D:VIDEOS*.*” “E:VIDEOS*.*” /D /S /C /Y >> e:Backup.txt
